# KAS

KAS is a private-deployment, industry-focused AIOps alert governance platform built on Keep OSS.

## Positioning

KAS is designed for organizations in Singapore that need a practical and controllable approach to alert operations.
It focuses on alert noise reduction, incident response acceleration, and workflow-driven operational delivery in private environments.

## What KAS Delivers

- Alert governance across the full lifecycle: ingestion, enrichment, correlation, routing, and closure.
- Private deployment options for enterprise-controlled environments.
- Industry-focused implementation patterns for regulated and operationally complex teams.
- AI-assisted analysis to improve triage quality and response speed.
- Workflow automation to connect alerts, notifications, ticketing, and execution actions.

## Focused Integration Scope

KAS is intentionally scoped to a focused closed-loop integration set:

- Alert ingestion: Prometheus
- Context enrichment: Grafana Loki
- AI analysis: OpenAI, DeepSeek, Ollama
- Notifications: SMTP, Webhook
- Ticketing: GitHub, Jira
- Automation executors: HTTP, SSH, Python

## Deployment Model

KAS is intended for private deployments with customer-owned operational control.
Typical deployment patterns include internal infrastructure, isolated environments, and controlled network boundaries.

## Platform Approach

KAS emphasizes:

- Fast implementation with a focused provider set
- Predictable operations with reduced integration sprawl
- Clear ownership between platform, SRE, and application teams
- Delivery-oriented automation for day-2 operations
